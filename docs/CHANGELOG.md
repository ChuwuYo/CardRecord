# 刷记 Release Notes

## Unreleased

### 新增

- 新建或编辑卡片时可选择未选择、借记卡或信用卡；历史卡片升级后保持未选择，等待用户自行补充。
- 主页增加借记卡与信用卡筛选；文件夹仍可无限创建，筛选面板会明确提示滚动并固定展示管理入口。
- 信用卡可选填每月账单日和还款日，并在详情页展示。
- 年费笔数按下次结算日前一个日历年统计；窗口尚未开始时不显示进度，也不允许误记有效笔数。
- 纯色与预设卡面均可选择卡组织；用户图片模式隐藏选择器但保留已保存的卡组织，横版、竖版与双列布局共享一致的品牌视觉规则。
- 用户选择的卡面图片会复制进应用私有存储；删除相册原图后，已保存卡片仍可正常显示。

### 修复

- 卡片主题色不再被卡组织覆盖；新增和再次编辑都会保留用户选色。
- 卡号、银行图标与文字使用一致的对比度保护；半透明徽标底板、卡组织水印和相交光环在浅色卡面上也保留轮廓。
- 长按删除增加不可撤销的二次确认，并明确会同时删除流水。
- 双列卡名、文字安全区、右下徽标与右上装饰重新排布，避免省略和重叠。
- 用户图片保持原图，只叠加文字；不再显示徽标、水印或额外装饰。
- 编辑保存改为非破坏性更新，保留历史流水；目标卡已删除时不会被意外重新创建。
- 修复部分旧版本升级后可能丢失文件夹关系或关联流水的问题。
- 卡片有效期按本地日历日判断，截止日当天不再提前显示过期。
- 文件夹删除后列表筛选自动回到有效状态；详情和编辑页正确区分加载、失败与卡片不存在。
- 全局操作提示会避让列表与详情页悬浮按钮；应用强制浅色/深色时，启动窗口与 Compose 使用同一明暗模式。

### 优化

- 备份升级为可见目录：始终包含 JSON，有图片时会创建图片文件夹；导入会自动匹配、校验并把图片复制回应用，成功后可删除导出目录。
- 导出和导入都会先检查备份完整性；内容变化、超出大小限制、图片缺失或损坏时，会在修改现有数据前停止并提示。
- 导入统一选择完整备份目录；覆盖与追加都会安全处理数据关联，导入导出过程可取消且不会并发执行。
- 用户卡面图片会在保存、放弃、删除、导入或取消后自动清理无用副本。
- 文件夹和流水较多时，列表加载与滚动更流畅。
- 关闭系统云备份与设备迁移，避免明文卡片数据被系统自动复制到其他位置。
- 改进英文数量文案和版本升级可靠性。

## v1.5.3 (2026-06-18)

## v1.5.2 (2026-06-17)

### 修复

- **自定义主题色现在会作用于整个界面**。此前选择自定义颜色时只有按钮和强调文字会变色，背景与卡面仍停留在默认的蓝绿色调；现已修复，背景、各级表面与容器颜色都会随所选颜色一同协调变化。

### 优化

- 自定义配色会生成协调的整套界面颜色，观感与系统动态色保持一致、过渡更自然。
